Программа генерации случайных паролей.
Автор: Янин Ярослав Иванович, группа МК-101

Опции командной строки:
  -minl <число>   Минимальная длина пароля
  -maxl <число>   Максимальная длина пароля
  -n <число>      Точная длина пароля 
  -c <число>      Количество паролей (по умолчанию 1)
  -a [строка]     Пользовательский алфавит (если передать опцию без строки, применится полный стандартный алфавит) 
  -C <a|A|D|S>    Наборы символов (a - строчные, A - заглавные, D - цифры, S - спецсимволы). Дубликаты наборов (например, aDa) запрещены!
  -d <символ>     Добавить разделитель значений к текущим (по умолчанию "=" и ":")
  -D <символ>     Заменить все текущие разделители на новые
  -p <числа>      Вероятности выпадения символов/наборов через запятую (например: -p 0.5,0.2). Оставшаяся вероятность распределяется между остальными символами равномерно.

Примечание: 
Значения опций можно указывать через пробел (например, -n 10) или слитно через любой активный разделитель (например, -n=10). Неизвестные опции (например, -m1, -s) парсер безопасно игнорирует.

Примеры запуска:
> my_program.exe -n=12 -C D -c 3
> my_program.exe -minl 8 -maxl 15 -a -c 5
> my_program.exe -n 50 -C aD -p 0.95
> my_program.exe -m3 -d! -n!10 -p -a -m2

СБОРКА И ТЕСТИРОВАНИЕ ПРОЕКТА (CMake)

1. ГЕНЕРАЦИЯ ПРОЕКТА
Создает нужные файлы для сборки в отдельной папке build. 
Выполнять из корневой директории проекта:
> cmake -B build

(Опционально для конкретной версии студии: cmake -B build -G "Visual Studio 17 2022" -A x64)
Проект можно открыть в Visual Studio через файл CMakeLists.txt или сгенерированный .sln.

2. СБОРКА ПРОЕКТА
Компиляция исходного кода. Выполнять из корня проекта:
> cmake --build build --config Debug
(Для релизной версии замените Debug на Release)

Запуск тестов (выполнять в папке build):
> cmake --build . --target test_all
> ctest -C Release
